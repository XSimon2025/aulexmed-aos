## Migration Steps

Nameserver cutover:

1. Confirm Cloudflare DNS contains the Vercel website records and Cloudflare Email Routing records.
2. At the current registrar/DNS provider, replace existing nameservers with:
   - `bristol.ns.cloudflare.com`
   - `skip.ns.cloudflare.com`
3. Wait for Cloudflare zone status to become `active`.
4. Re-check Vercel domain status for `aulexmed.com` and `www.aulexmed.com`.
5. Re-check Cloudflare Email Routing status and enable if Cloudflare still reports `unconfigured`.
6. Test:
   - `https://aulexmed.com`
   - `https://www.aulexmed.com`
   - `support@aulexmed.com`
   - `sales@aulexmed.com`
   - `b2b@aulexmed.com`

Post-cutover:

1. Send a test inquiry email from the website after contact workflows are connected.
2. Confirm DKIM and SPF pass in the recipient inbox message details.
3. Keep `MAIL_SETUP.md` and this document updated after any DNS, email, or hosting change.
