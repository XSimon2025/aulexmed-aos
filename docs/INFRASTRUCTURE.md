Security notes:

- Keep service-role usage server-side only.
- Review RLS policies before exposing any new public tables.

## Migration Steps

Nameserver cutover:

1. Confirm Cloudflare DNS contains the Vercel website records and Cloudflare Email Routing records.
2. At the current registrar/DNS provider, replace existing nameservers with:
   - `bristol.ns.cloudflare.com`
   - `skip.ns.cloudflare.com`
3. Wait for Cloudflare zone status to become `active`.
4. Re-check Vercel domain status for `aulexmed.com` and `www.aulexmed.com`.
5. Re-check Cloudflare Email Routing status and enable if Cloudflare still reports `unconfigured`.
6. Test:
   - `https://aulexmed.com`
   - `https://www.aulexmed.com`
   - `support@aulexmed.com`
   - `sales@aulexmed.com`
   - `b2b@aulexmed.com`

Post-cutover:

1. Add Resend DNS records when transactional sending is ready.
2. Add Resend environment variables in Vercel.
3. Send a test inquiry email from the website after Resend is integrated.
4. Keep `MAIL_SETUP.md` and this document updated after any DNS, email, or hosting change.
